Oliver Pryce
Founder, SketchCAD

I built SketchCAD because I kept having 3D ideas I couldn't quickly get out of my head. Traditional CAD software felt like a barrier, not a tool. I wanted something that felt as fast as sketching on paper but produced real geometry you could actually use.

SketchCAD started as an experiment to see how well language models could generate 3D geometry. After testing 9 models across 46 design prompts, the results were better than I expected, and I decided to share it.

Under the hood

For the technically curious.

You don't need any of this to use SketchCAD. But if you want to know how it works at a deeper level, here it is.

Signed Distance Functions (SDFs)

Instead of building geometry with polygons, SketchCAD uses mathematical functions that describe shapes implicitly in 3D space. This makes them GPU-native, highly composable, and suited to organic or procedural geometry that traditional B-Rep CAD cannot produce.

geometry representation

LLM geometry generation via WGSL

The language model outputs a single WGSL shader function: fn map(p: vec3f) -> f32. Frontier models already understand SDF syntax from pre-training on Inigo Quilez's widely referenced tutorials. SketchCAD exploits this prior directly, with no fine-tuning required.

AI + shaders

Grammar-constrained DSL fallback

For weaker models, SketchCAD falls back to a custom DSL compiled by a C++ geometry kernel. Grammar-constrained decoding enforces structure at inference time, eliminating syntax errors on models that struggle with raw WGSL output.

reliability layer 
// From: "a gyroid lattice with smooth sphere nodes"

fn map(p: vec3f) -> f32 {
  let q = p * 1.5;

  // gyroid field
  let g = sin(q.x)*cos(q.y)
        + sin(q.y)*cos(q.z)
        + sin(q.z)*cos(q.x);

  let gyroid = abs(g) - 0.3;

  // repeated spheres at nodes
  let cell = fract(p * 0.5) - 0.5;
  let sphere = length(cell) - 0.18;

  return opSmoothUnion(gyroid, sphere, 0.1);
}
